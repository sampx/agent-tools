# 创建 PRD: 新产品需求文档

## 使用方式

```bash
/create-prd                    # 交互式输入产品名称
/create-prd my-new-product     # 指定产品名称
```

## 定位

**仅用于新产品**。现有项目的新功能请使用 `/plan-feature`。

## 流程

### 1. 确定产品名称

- 从参数获取，或询问用户
- 输出路径：`docs/PRD-{product-name}.md`

### 2. 收集上下文

**必读：**
- `MEMORY.md` - 获取工作空间架构经验
- `.agents/templates/PRD-template.md` - PRD 模板

**可选：**
- 如果有参考资料或讨论历史，从中提取需求

### 3. 生成 PRD

基于对话上下文和模板生成完整 PRD：
- 填充所有必选章节
- 小型项目可简化，聚焦核心章节（1-5, 8, 11, 12）
- 标注因信息缺失而做出的假设

### 4. 质量检查

- [ ] 所有必选章节已填写
- [ ] MVP 范围现实且定义清晰
- [ ] 技术选型有合理依据

### 5. 输出确认

1. 确认文件路径：`docs/PRD-{product-name}.md`
2. 简要总结 PRD 内容
3. 建议后续：评审后可使用 `/plan-feature` 创建实现计划
