# Prime: 加载上下文

## 目标

唤醒 Wopal，建立对工作空间的全景认知，快速进入就绪状态。

> **注意**：本命令聚焦于**空间级认知**。如需深入某个子项目开发，请按 AGENTS.md 中的"上下文加载协议"加载该子项目的 `AGENTS.md`。

## 流程

### 0. 唤醒 Wopal

- 如果尚未加载 Wopal 身份和记忆（SOUL.md、USER.md、MEMORY.md），则执行唤醒流程
- 如果已加载，跳过此步骤

### 1. 建立空间认知

- 读取 `.workspace.md` 获取工作空间目录全景地图（无论是否已唤醒，每次 prime 都应读取）

### 2. 恢复短期记忆

- 读取 `memory/` 目录下当日和前日的短期记忆文件（先列目录，无则跳过）

### 3. 了解当前状态

查看当前分支和未提交变更：
!bash git status

查看最近的提交记录：
!bash git log -5 --oneline

## 输出报告

提供简洁的总结，包括：

### 🧙‍♀️ Wopal 唤醒状态
- 记忆要点（从 MEMORY.md 和最近 memory/ 提取的关键信息）
- 准备就绪确认

### 当前状态
- 当前分支
- 最近的变更或开发重点
- 未提交的变更（如有）
- 短期记忆中的待办事项（如有）

**确保总结易于快速浏览，使用项目符号和清晰的标题。**
